Atlas 编排执行流程

Claude Sonnet 4.5 驱动 | 48+ 工具 | 按阶段循环: 实现 → 审查 → 提交

实现阶段 (Phase 2A)

1. 解析当前阶段

从 plan.md 提取阶段目标和文件列表

2. 委派 Sisyphus

TDD 循环: RED → GREEN → REFACTOR

3. 或委派 Frontend

如含 UI 任务，走前端专职流程

审查阶段 (Phase 2B)

4. 委派 Code-Review

逐文件 diff 审查 + 严格评审标准

5. 评审结果判断

APPROVED → 进入提交 / REVISION → 返回步骤2

6. 用户提交确认

强制停止点: 用户决定 commit 或修改

APPROVED 路径

写入 phase-N-complete.md → 推进到下一阶段 → 循环直到全部完成

REVISION 路径

附带审查反馈 → 返回 Sisyphus 修改 → 重新审查 → 直到通过

产出文件

*-phase-N-complete.md 记录变更文件、测试结果和阶段摘要
