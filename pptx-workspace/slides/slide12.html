11 / DESIGN PRINCIPLES

六大设计原则

构建可靠、可扩展、高效的 AI 多 Agent 协作系统

1. 单一职责

Single Responsibility

  • 每个 Agent 专注一件事
  • Explorer 只探索不实现
  • Sisyphus 只实现不审查
  • 职责零重叠

2. 最小权限

Least Privilege

  • 工具按需分配
  • Explorer 没有写权限
  • Code-Review 无编辑能力
  • 防止越权操作

3. TDD 驱动

Test-Driven Development

  • RED: 先写失败测试
  • GREEN: 最少代码通过
  • REFACTOR: 清理优化
  • 强制每阶段执行

4. 人在环中

Human-in-the-Loop

  • 计划必须用户审批
  • 每阶段需确认提交
  • 最终完成需用户确认
  • 3 个强制停止点

5. 上下文保护

Context Conservation

  • 委派重型操作
  • 返回精简摘要
  • 主 Agent 保持轻量
  • 节省 70-80% 窗口

6. 可扩展性

Extensibility

  • 标准 .agent.md 格式
  • 轻松添加新 Agent
  • 自定义工具和模型
  • 插件化架构
