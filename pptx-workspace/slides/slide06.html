05 / CORE WORKFLOW

核心工作流程

完整生命周期：Planning → Implementation → Review → Commit

Phase 1

规划

  • 分析需求
  • 委派 Explorer 探索
  • 委派 Oracle 研究
  • 撰写 3-10 阶段计划
  • 等待用户审批

Phase 2

实现循环

  • 2A: 委派实现
  • 2B: 委派审查
  • 2C: 返回用户提交
  • 循环每个阶段
  • REVISION 则回退

Phase 3

完成

  • 编写完成报告
  • 汇总所有文件变更
  • 确认测试全部通过
  • 呈现最终摘要
  • 关闭任务

3 个强制停止点

计划审批 → 每阶段提交确认 → 最终完成确认

TDD 驱动每阶段

RED (写测试失败) → GREEN (最少代码) → REFACTOR (优化)

文件输出规范

*-plan.md → *-phase-N-complete.md → *-complete.md
